这两个红色盒子原则上都会向前查找到div标签来定位
而第二个红色盒子因为后渲染导致把第一个红色盒子in挤到下面了
在默认渲染顺序下，第二个绝对定位盒子通常会显示在第一个绝对定位盒子的前面，而不是后面。
这是因为在 HTML 文档中，浏览器按照元素出现的先后顺序进行渲染。
当两个绝对定位元素的z-index值相同时（都为默认的auto）
后渲染的元素会在视觉上覆盖先渲染的元素
即后出现的元素会显示在上面，看起来像是在前面。
“绝对定位盒子 out” 在 HTML 文档中位于 “绝对定位盒子 in” 之后
所以在默认情况下，“绝对定位盒子 out” 会覆盖在 “绝对定位盒子 in” 的前面，而不是在其后面。
